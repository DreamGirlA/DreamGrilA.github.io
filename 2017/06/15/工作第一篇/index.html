VUE总结

生命周期



  1. beforeCreate 创建前状态 ,el 和 data 并未初始化。

    可以在这加个loading事件

  2. created:created 创建完毕状态, 完成了data数据的初始化，el没有

    在这结束loading，还做一些初始化，实现函数自执行 。

  3. beforeMount 挂载前状态, 完成了el和data初始化。

    这时el还是，应用的虚拟Dom技术，先把坑占住了。到后面mounted挂载的时候再把值渲染进去。

  4. mounted 挂载结束状态, 完成挂载。

    在这发起后端请求，拿回数据，配合路由钩子做一些事情

  5. beforeUpdate 更新前状态。

    data里的值被修改后，将会触发update的操作

  6. updated 更新完成状态

  7. beforeDestroy 销毁前状态。

    你确认删除XX吗？ destoryed ：当前组件已被删除，清空相关内容

  8. destroyed 销毁完成状态。

    销毁完成后，我们再重新改变message的值，vue不再对此动作进行响应了。但是原先生成的dom元素还存在，可以这么理解，执行了destroy操作，后续就不再受vue控制了。

计算属性computed: {} vs Methods methods: {}

计算属性是基于它们的依赖进行缓存的。计算属性只有在它的相关依赖发生改变时才会重新求值。这就意味着只要 message 还没有发生改变，多次访问 reversedMessage 计算属性会立即返回之前的计算结果，而不必再次执行函数。

相比而言，只要发生重新渲染，method 调用总会执行该函数。

我们为什么需要缓存？假设我们有一个性能开销比较大的的计算属性 A ，它需要遍历一个极大的数组和做大量的计算。然后我们可能有其他的计算属性依赖于 A 。如果没有缓存，我们将不可避免的多次执行 A 的 getter！如果你不希望有缓存，请用 method 替代。

Computed 属性 vs Watched 属性

Vue 确实提供了一种更通用的方式来观察和响应Vue实例上的数据变动：watch属性。当你有一些数据需要随着其它数据变动而变动时，你很容易滥用 watch——特别是如果你之前使用过 AngularJS。然而，通常更好的想法是使用 computed 属性而不是命令式的 watch 回调。

父组件与子组件进行通信

  1. 在父组件parent.js中通过ajax请求到了数据，需要将数据传到前端，在data定义你要传的值XX，在函数中用this.XX = 数据库的值。

  2. 在html中，接收传递的数据名称不能大写，v-bind:xxxx=’XX’,这时xxxx就是我们可以使用的数据了。

  3. 在子组件child.js中，props:[‘articledata’],//接收父组件传过来的数据

    export default {
      data() {
          return {
              titleurl:[],//详情页面地址,前端要直接使用的
          }
      },
      props:['articledata'],//接收父组件传过来的数据
      watch:{
         articledata:'getDataSuccess' //监听数据变化
      },
      methods:{
        getDataSuccess:function(){
            // 获取本地地址
              let whref = window.location.href;
              whref = whref.slice(0,-10);
            for(let i =0;i < this.articledata.length;i++){
                // 为了点击标题后跳转页面，拼接url
                this.titleurl[i] = whref + 'articledetail.html?article_id=' + this.articledata[i].id;
            }
        },
      timeformat : format // format是个函数，可以直接将函数赋给它
    }
}

    div class="am-g" v-for="(item,index) in articledata" v-if="articledata"

    img :src="imagesrc[index]"//直接索引对应图片地址
a :href="titleurl[index]" target="_blank">{{item['title']}}/a
